Overview

This deposit contains two fully shared R Markdown scripts and example output
rasters for small area estimation of under-five mortality in Ethiopia at 5 km resolution.
Important note on Z-score statistics
The covariate rasters in this deposit were standardised using global GBD reference
means and standard deviations before masking to Ethiopia. Consequently, the
Ethiopia-only mean is not zero and the standard deviation is not one.
Relative comparisons and spatial patterns within Ethiopia are still valid. For 
absolute z-score interpretation (e.g., percentiles), re-standardise using each
raster's own mean and SD (provided in the accompanying JSON metadata files).
Covariates derived from satellite/ERA5 (NDVI, Runoff, etc.) were standardised 
after masking and have correct z-score properties (mean near 0, SD near 1).


1. Scripts
## `covariate_adjustment.Rmd`

* Processes and harmonizes 25 candidate raster covariates used for under-five mortality 
modelling in Ethiopia.
* Downloads and prepares yearly raster datasets from multiple sources, including 
ERA5, GBD, MODIS, WorldPop, Malaria Atlas Project, and related geospatial products.

* Covariate availability differs by source:

  * some variables are available for 2000–2016,
  * some extend to 2017_2023,
* Whenever source data were publicly available, all years provided by the 
original data source were processed and standardized.

* Applies consistent preprocessing steps to all raster layers, including:

  * reprojection,
  * resampling to a common 5 km grid,
  * masking to Ethiopia,
  * log1p transformation (where appropriate),
  * z-score standardization,
  * and capping of extreme values.
* Under five population count rasters and inhabited area masks were processed using the same 
spatial harmonization workflow. These layers were prepared annually for 2000–2019.
* Exports all processed covariates, Under five populations, and inhabited areas as GeoTIFF rasters at 5 km spatial resolution.
* Each exported raster is accompanied by a JSON metadata file documenting:

  * source information,
  * temporal coverage,
  * preprocessing steps,
  * transformation methods,
  * and summary statistics.

## `censoring_adjustment.Rmd`

* Uses the adjusted raster outputs generated by `covariate_adjustment.Rmd`
together with Ethiopia DHS child-level data.
* Starts from the full set of 25 adjusted covariates.
* Evaluates pairwise Pearson correlations across covariates and removes 7 highly
correlated variables (|r| > 0.7), retaining 18 uncorrelated covariates for 
downstream modelling.
* Integrates these 18 spatial covariates with DHS-derived demographic predictors, including:

  * household wealth,
  * birth plurality,
  * maternal age at birth,
  * sex of child,
  * and related survey variables.
* Prepares survival datasets for children under five years of age at the time of 
survey (informative censoring framework).
* Evaluates three survival modelling approaches for censoring adjustment:

  * Cox proportional hazards (CoxPH),
  * Random Survival Forest (RSF),
  * Gradient Boosting Machine (GBM).
* Model performance is assessed using:

  * concordance index (C-index),
  * area under the ROC curve (AUC),
  * and comparison against external mortality benchmarks.
* Across all evaluated years, the RSF model consistently achieved the best predictive 
performance and was therefore selected as the final censoring-adjustment model.
* The script produces:

  * adjusted mortality estimates,
  * survival probabilities,
  * model evaluation summaries,
  * and intermediate datasets used in downstream spatial modelling.
* Summary evaluation tables are provided in the accompanying Excel files included 
in this repository.


## Third Script: Final INLA+SPDE Model (Not Publicly Released)

* Uses the outputs generated from both:

  * `covariate_adjustment.Rmd`
  * and `censoring_adjustment.Rmd`.
* Starts from the 18 uncorrelated covariates retained after correlation filtering.
* Performs additional variable selection using:

  * Variance Inflation Factor (VIF),
  * correlation with mortality,
  * and model performance criteria.
* Retains 17 covariates for the final Bayesian spatial-temporal model.
* Fits annual small-area estimation models using Bayesian inference with:

  * Integrated Nested Laplace Approximation (INLA),
  * and Stochastic Partial Differential Equation (SPDE) spatial modelling.
* Produces annual 5 km resolution maps of:

  * under-five mortality rate,
  * uncertainty intervals,
  * and estimated death counts.
* The complete implementation of the final INLA+SPDE modelling workflow is not 
publicly released in this repository.
* Example outputs, validation summaries, metadata, and processed covariate rasters 
are provided to support transparency and reproducibility.

## Covariate preprocessing

Twenty‑five raster covariates (2000‑2015) were standardised to consistent names
and merged into a single data frame. For each year, pixel values were extracted 
using a reference grid, column names were renamed (e.g., `stunting_2000...` → `Stunting`),
a `year` column was added, and the result was saved. All yearly files were then 
row‑bound. The final dataset contains 45,931 pixels × 16+ years × 25 covariates. 

## Covariate Summary Statistics and Transformation Assessment

To support transparency, reproducibility, and preprocessing decisions,
this repository includes a summary table describing the statistical
properties of all raster covariates used in the analysis workflow.

The table reports, for each covariate:

global mean and median values,
minimum and maximum raster values,
skewness,
percentage of zero-valued pixels,
recommended preprocessing or transformation strategy,
and temporal coverage of the raster dataset.

Transformation recommendations were based on distributional characteristics 
observed across all yearly rasters. Variables with strong right-skewness or 
high zero inflation were evaluated for possible log1p or square-root 
transformation prior to standardization and modelling.

The summary table was generated to:

document covariate behaviour before modelling,
support reproducible preprocessing decisions,
identify highly skewed variables,
evaluate zero inflation,
and improve interpretation of the spatial covariate layers.

This table does not contain any restricted DHS information or geographic 
identifiers and is therefore publicly shareable through GitHub and Zenodo.

Included file:

`covariate_summary_statistics.csv`


## Cheeck multicolinearity

A pairwise correlation matrix was computed for all 25 covariates across all pixels 
and years (2000‑2015). Variables with absolute Pearson correlation > 0.7 were 
identified using `caret::findCorrelation`. Highly correlated variables were removed, 
keeping a subset with |r| ≤ 0.7.

##  DHS Processing, Covariate Integration, and Censoring Preparation Introduction

This module prepares the DHS child-level dataset for downstream modelling by harmonizing 
survey variables, integrating geospatial covariates, and applying consistent censoring rules. 
It serves as the bridge between earlier raster covariate preparation and subsequent 
prediction and spatial modelling steps (e.g., INLA + SPDE).

The goal of this stage is to produce a clean, analysis-ready survival dataset that
aligns demographic, temporal, and spatial information at the individual level.

1. DHS Data Harmonization and Outcome Construction

Input:

Merged DHS dataset (.dta)

Processing steps:

Construct age in months (b19) using available DHS variables (b7, b8)
Convert Ethiopian calendar to Gregorian (country-specific adjustment; modify if applied elsewhere)
Create unique cluster-year identifier (Clu_id) using cluster (v001), survey year, and birth year
Recode survival outcome:
b5 = 1 → death (event)
b5 = 0 → alive (censored)
Restrict dataset to births from 2000 onward (to align with covariate availability)

Output:

Sample size summary by birth year (Summary_data_processing.csv)

2. Spatial Covariate Linkage (5 km Grid)

This step links DHS clusters to previously prepared raster covariates.

Inputs:

DHS cluster coordinates (WGS84)
5 km raster covariate grid (18 selected variables)

Processing steps:

Convert DHS clusters to spatial features
Convert DHS cluster coordinates (latitude and longitude) into point data and overlay them with the covariate raster layers
Extract values from all 18 covariate rasters for each DHS cluster point
Merge the extracted raster covariate values with all child records belonging to the corresponding cluster

Output:

DHS_2000_with_covariates.rds (stored locally due to DHS data sharing restrictions; not publicly shared)


3. Covariate Cleaning and Missing Data Handling

Processing steps:

Ensure all raster covariates are in numeric format
Impute key continuous covariates (e.g., NDVI, slope) using median values where required (if the missing is happen during marge)
Remove remaining incomplete cases across spatial covariates to ensure consistency
Preserve DHS structural missingness where appropriate (e.g., survival-related variables)

Output:

Cleaned DHS–covariate dataset ready for survival processing


4. Survival Data Construction and Censoring Adjustment

This step standardizes time-to-event definitions for under-five mortality analysis.

Input:

Import cleaned data

Processing steps:

Define survival time using age in months (b7)

Apply censoring rule:

Children aged ≥ 60 months → treated as censored (alive)
Assign missing age values to exceed the under-five threshold where necessary
Ensure consistency between age and survival status
Retain modelling variables:
Outcome (b5)
Time (b7)
Demographic variables (e.g., wealth, sex)
18 spatial covariates

5. RSF-Based Censoring Adjustment (Intermediate Output)

This component estimates mortality risk among censored observations using a 
Random Survival Forest (RSF) approach.

Processing steps:

Fit RSF model using demographic and spatial predictors
Predict individual survival probabilities
Estimate conditional probability of death before age five for censored children
Aggregate predicted risks to adjust mortality estimates
Generate bootstrap-based uncertainty intervals

Outputs:

Predicted survival probabilities (individual-level)
Adjusted mortality estimates 
Intermediate datasets for validation and comparison

Notes for Reproducibility

File paths should be updated to match the local environment
Ethiopian calendar conversion is context-specific and should be modified for other countries
Variable names follow DHS standard recode format; adjust if using alternative survey sources
This module assumes covariates are already preprocessed and aligned to a 5 km grid


Module Output Summary

This stage produces:

Harmonized DHS dataset
DHS dataset linked with spatial covariates
Survival-ready dataset with censoring applied
Intermediate model outputs (RSF-based adjustment)

These outputs are intended for:

Sharing code  (GitHub / Zenodo)
Feeding into final prediction workflows (not included here)

This module is designed as a standalone, reproducible processing step that

suitable for sharing and validation.

A pairwise correlation matrix was computed for all 25 covariates across all pixels 
and years (2000‑2015). Variables with absolute Pearson correlation > 0.7 were 
identified using `caret::findCorrelation`. Highly correlated variables were 
removed, keeping a subset. The reduced correlation matrix was saved 
as `reduced_correlation_selected.csv` and visualised in `correlation_plot_selected.tiff` (attached).
The final list of selected covariates is stored in `selected_covariates_list.txt`. Code: `R/02_select_covariates.R`.

## Covariate selection: remove highly correlated variables (|r| > 0.7)
- Load all yearly data (2000-2015) and combine into one data frame
- Convert all covariate columns to numeric and remove rows with NA
- Compute Pearson correlation matrix for 25 covariates
- Use caret::findCorrelation(cutoff = 0.7) to identify variables to drop
- Keep only uncorrelated covariates (|r| ≤ 0.7)

### Covariate Selection for Final Model

Starting from the **18 uncorrelated covariates** identified after correlation 
filtering, an additional selection step was performed within the final INLA model.
This process retained **17 covariates** for the final model.
Only **night-time lights** was excluded.

**The 17 covariates used in the final INLA+SPDE model are:**

1. Diarrhea incidence
2. Dew point temperature
3. DPT3 vaccination coverage
4. Education coverage
5. WASH
6. Improved water coverage
7. NDVI
8. Lower respiratory infection
9. Slope
10. Stunting prevalence
11. HIV prevalence
12. Malaria prevalence
13. Solar radiation
14. PM2.5
15. Potential evaporation
16. Runoff
17. Severe wasting prevalence

**Excluded:** Night-time lights

The final Bayesian spatial-temporal INLA + SPDE model was fitted independently 
for each survey year from 2000 to 2015. The accompanying table summarizes model 
validation metrics derived from training, validation, and independent test
datasets for each year.

Model performance was evaluated using predictive coverage, bias correction, 
optimal tuning parameters, and comparison of weighted mortality estimates 
between training and validation datasets. Overall, the models demonstrated 
stable and reliable predictive performance across survey years, supporting
their use for national and subnational spatial estimation and mapping of 
under-five mortality.

Coverage represents the proportion of held-out test observations captured within
the 95% credible intervals. Most survey years achieved coverage values close to
the nominal 95% target, indicating appropriate uncertainty calibration and 
robust predictive performance.

The weighted mean represents the population-weighted under-five mortality rate 
(U5MR) estimated from DHS observations, while the training rate corresponds to 
the mortality rate calculated from the training subset used during model fitting. 
Bias correction values indicate the post-validation calibration adjustment 
applied to improve agreement between predicted and observed mortality estimates.
The optimal k parameter corresponds to the selected tuning parameter used during 
model optimization and validation.

Important Year-Specific Note (2002)

The 2002 model showed comparatively lower test coverage relative to most other 
survey years. This difference may reflect temporal and spatial variability in 
survey cluster distribution and regional heterogeneity affecting prediction 
uncertainty. Nevertheless, model performance remained acceptable for national 
and subnational spatial estimation and mapping applications.

Interpretation Notes
Coverage = proportion of test observations falling within the 95% credible intervals.
Bias correction = calibration adjustment applied after validation.
Optimal k = selected tuning parameter during model optimization.
Weighted mean = population-weighted under-five mortality rate per child.
Training rate = observed mortality rate in the training dataset.

Data Sharing Restrictions

This repository does not include restricted DHS microdata, household identifiers, or cluster-level geographic coordinates.

All DHS-derived analyses were conducted under the DHS Program data-use agreement. Any datasets generated through direct linkage of DHS records with spatial covariates remain restricted and are not publicly distributed.

Only aggregated outputs, synthetic example data (`synthetic_dhs.rds`), derived raster products, metadata, and reproducible preprocessing resources are shared in this repository.
